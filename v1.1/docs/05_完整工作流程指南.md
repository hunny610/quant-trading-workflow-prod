## 🎯 系統概述

本系統採用 **混合模式（Hybrid Mode）**，結合：
- **Blave 平台** - 深度結構化數據（專業數據）
- **網路搜索** - 最新即時資訊（廣度數據）
- **TradingView** - Volume Profile（技術分析）

---

## ✨ 最佳實踐：混合模式

### 為什麼混合模式？

| 數據來源 | 優點 | 適用場景 |
|---------|------|---------|
| **Blave** | 深度、結構化、獨特指標 | 籌碼集中度、巨鯨追蹤、OI失衡 |
| **網路搜索** | 最新、即時、廣度 | 最新價格、新聞、市場情緒 |
| **TradingView** | Volume Profile 權威 | POC/VAH/VAL 技術位 |

### 混合模式分工

#### Blave 負責（深度數據）
1. ✅ 籌碼集中度 - 持有者分佈
2. ✅ 巨鯨警報 - 大戶動向
3. ✅ Blave 頂尖交易員 - 聰明錢追蹤
4. ✅ OI 失衡 - 合約數據
5. ✅ 資金費率 - 永續合約
6. ✅ 多空力道 - 買賣壓力
7. ✅ 爆倉數據 - 清算熱圖
8. ✅ 市場情緒 - 整體氛圍

#### 網路搜索負責（最新資訊）
1. ✅ 最新價格 - BTC/ETH/SOL 等即時市價
2. ✅ 最新新聞 - 市場驅動因子
3. ✅ 經濟行事曆 - 總體經濟數據
4. ✅ 熱門板塊 - 當前趨勢
5. ✅ 技術分析共識 - 市場觀點匯總
6. ✅ 地緣政治風險 - 宏觀事件

## 🚀 完整工作流程

### 階段 1：準備工作

確認您在這個目錄：
```
自動交易提示字自動化工作流程
```

確保已安裝依賴：
```powershell
cd quant-trading-workflow-test\20250506
npm install
```

### 階段 2：【您手動執行】Blave 數據抓取（選擇性）

如果需要深度數據，您可以選擇以下任一方式執行 Blave 任務：

**選項 A：批次抓取（推薦！最簡單）
```powershell
# 在根目錄執行（自動交易提示字自動化工作流程）
.\Blave批次抓取.ps1
```

**選項 B：個別任務執行（更穩定）
```powershell
# 先切換到正確目錄
cd quant-trading-workflow-test\20250506

# 高優先級任務（建議先執行）
npm run capture:blave:toptrader:position
npm run capture:blave:holder_concentration:overview
npm run capture:blave:whale_hunter:overview

# 中優先級（可選）
npm run capture:blave:market_sentiment:overview
npm run capture:blave:taker_intensity:overview
npm run capture:blave:oi_imbalance:overview
npm run capture:blave:liquidation:overview

# 低優先級（可選）
npm run capture:blave:funding_rate:overview
npm run capture:blave:funding_rate:history
npm run capture:blave:squeeze_momentum:history
```

**選項 C：跳過 Blave 抓取，只用網路搜索
- 如果 Blave 抓取一直失敗，可以跳過 Blave 數據
- 直接進入「階段 3：網路搜索」
- 提示詞仍可正常執行，Blave 數據會標註 N/A

### 階段 3：網路搜索（必備）

使用 AI 助手進行以下搜索：

#### 搜索 1：最新價格
```
BTC ETH SOL XRP BNB price today YYYY-MM-DD cryptocurrency market
```

#### 搜索 2：市場新聞
```
cryptocurrency market news today macro economic events YYYY-MM-DD
```

#### 搜索 3：技術分析
```
Bitcoin Ethereum support resistance levels current analysis YYYY
```

#### 搜索 4：Volume Profile（可選）
```
Bitcoin Ethereum volume profile POC VAH VAL current levels YYYY
```

### 階段 4：整合分析

結合 Blave 數據 + 網路搜索結果，按照 [交易提示詞模板](#交易提示詞模板) 生成分析報告。

## ⚠️ 注意事項

1. **Blave 需要登入** - 首次使用需手動登入 Blave
2. **網路搜索最新優先** - 永遠優先使用網路搜索獲取最新價格
3. **數據時間一致性** - 確保 Blave 數據與網路搜索時間接近
4. **風險控制** - 永遠設置止損，不要過度槓桿
5. **驗證數據** - 交叉核對多個來源的數據一致性

## 📞 故障排除

### 問題 1：Blave 無法擷取
- 確認已登入 Blave
- 檢查網路連線
- 手動打開 Blave 頁面確認正常

### 問題 2：中文路徑錯誤
- 使用根目錄的 `Blave批次抓取.ps1` 腳本
- 避免在子目錄執行

### 問題 3：網路搜索資訊不足
- 調整搜索關鍵字
- 增加搜索來源數量
- 交叉核對多個新聞網站
