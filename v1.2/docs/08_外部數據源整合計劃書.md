## 📅 整合實施計劃（分階段）

### 階段 1：Nansen 整合（優先級 🔴 最高，預計 2-3 週）

**目標**：補足 Blave 在 Smart Money 和錢包標記方面的盲點

**任務清單**：
1. 研究 Nansen API 文檔
2. 設計數據抓取模組（類似 blave_capture.mjs）
3. 整合以下數據：
   - Smart Money 錢包列表
   - 標記錢包（VC、ETF、機構）
   - 敘事追踪（Memecoin、DeFi、AI）
4. 創建 npm scripts（如 `capture:nansen:smartmoney`）
5. 更新提示詞以支援 Nansen 數據

**預期產出**：
- `src/nansen_capture.mjs`
- `data/nansen/` 目錄結構
- 整合後的提示詞模板

## ⚠️ 風險與注意事項

1. **API 付費限制**：大部分平台的高級數據需要付費
2. **數據一致性**：不同平台數據定義可能不同，需要標準化
3. **抓取速率限制**：注意 API rate limit，避免被封鎖
4. **Blave 登入依賴**：新增平台不需要 Blave 登入
5. **提示詞更新**：每次整合新平台都需要更新提示詞模板
