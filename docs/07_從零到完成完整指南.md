## 🎯 專案概述

### 這是什麼？

這是一個**量化交易分析輔助系統**，透過自動化方式抓取市場數據，結合 AI 分析，產生穩健的交易建議。

### 核心特色

| 特色 | 說明 |
|-----|------|
| **混合模式** | 結合 Blave 深度數據 + 網路搜索最新資訊 |
| **嚴格風控** | R:R ≥ 1:3、清算 1% 迴避帶、POC 一致性檢查 |
| **您手動執行抓取** | 您自己選擇並執行 Blave 數據抓取任務 |
| **多種提示詞** | 穩健版 / 高盈虧版 / 全方位分析 可選 |

### 為什麼需要這個？

- **解決數據分散問題**：數據來自多個平台，手動收集耗時
- **解決資訊過載問題**：AI 幫你篩選真正重要的資訊
- **解決情緒交易問題**：嚴格的數據驅動流程，避免衝動決策

## 🚀 第一次設置：從零開始

### 階段 1：確認環境需求

#### 1.1 硬體與軟體需求

| 項目 | 最低要求 | 建議規格 |
|-----|---------|---------|
| 作業系統 | Windows 10 | Windows 11 |
| 處理器 | 雙核心 | 四核心以上 |
| 記憶體 | 4GB | 8GB+ |
| 硬碟空間 | 2GB | 10GB+ |
| 瀏覽器 | Microsoft Edge | 最新版 Edge/Chrome |

#### 1.2 安裝必要軟體

##### 步驟 1.2.1：安裝 Node.js

1. 開啟瀏覽器，前往 [https://nodejs.org/](https://nodejs.org/)
2. 下載 **LTS 版本**（長期支援版）
3. 執行安裝程式，一路按「下一步」
4. 安裝完成後，開啟 PowerShell 驗證：

```powershell
node --version
npm --version
```

應該會看到版本號（例如 `v20.x.x`）

##### 步驟 1.2.2：確認 Microsoft Edge 已安裝

- Windows 10/11 預設已安裝 Edge
- 如果沒有，從 [https://www.microsoft.com/edge](https://www.microsoft.com/edge) 下載安裝

### 階段 3：首次 Blave 登入設定

#### 3.1 為什麼需要登入？

- Blave 是付費平台，需要登入才能存取數據
- 系統使用瀏覽器的 persistent profile，登入狀態會保留在本機
- 帳密資訊**不會**上傳或儲存到任何地方

#### 3.2 首次登入步驟

##### 步驟 3.2.1：準備 Blave 帳號

確認您有 Blave 帳號且可以正常登入：
- 網址：https://blave.org/
- 若沒有帳號，請先註冊

##### 步驟 3.2.2：執行首次登入腳本

1. 在 PowerShell 中執行：

```powershell
cd d:\自動化查詢幣種專案
npm run capture
```

2. 這會開啟 Edge 瀏覽器
3. 在瀏覽器中手動登入 Blave
4. 登入成功後，回到 PowerShell 按 **Enter** 繼續
5. 腳本會執行一次數據抓取測試
6. 完成後，您的登入狀態已保存到 `.profile` 目錄

##### 步驟 3.2.3：確認登入成功

檢查 `.profile` 目錄是否已建立：

```powershell
Test-Path .profile
```

應該顯示 `True`

### 階段 4：第一次完整測試

#### 4.1 測試完整流程

現在讓我們執行一次完整的測試，確認一切正常：

##### 步驟 4.1.1：清空舊數據（重要！）

```powershell
npm run clear
```

應該看到「🧹 已清空舊的輸出目錄」

##### 步驟 4.1.2：【您手動執行】Blave 數據抓取

您可以選擇以下任一方式：

**選項 A：批次抓取（快速，但可能失敗）**
```powershell
.\scripts\抓取數據腳本\blave_all.ps1
```

**選項 B：個別任務執行（推薦，更穩定）**
```powershell
# 高優先級任務（建議先執行）
npm run capture:blave:toptrader:position
npm run capture:blave:holder_concentration:overview
npm run capture:blave:whale_hunter:overview

# 中優先級（可選）
npm run capture:blave:market_sentiment:overview
npm run capture:blave:taker_intensity:overview
npm run capture:blave:oi_imbalance:overview
npm run capture:blave:liquidation:overview

# 低優先級（可選）
npm run capture:blave:funding_rate:overview
npm run capture:blave:funding_rate:history
npm run capture:blave:squeeze_momentum:history
```

**選項 C：跳過 Blave 抓取，只用網路搜索**
- 如果 Blave 抓取一直失敗，可以跳過 Blave 數據
- 直接使用網路搜索進行分析
- 提示詞仍可正常執行，Blave 數據會標註 N/A

期間您會看到：
- 瀏覽器自動開啟
- 逐個造訪 Blave 頁面
- 進度顯示（例如：Task 1/10 completed）

##### 步驟 4.1.3：確認完成

當看到「All tasks completed! (100%)」時，表示成功！

按 Enter 退出。

##### 步驟 4.1.4：檢查輸出

確認 `data/blave/` 目錄下有數據：

```powershell
Get-ChildItem data\blave\
```

應該看到多個 `batch_YYYY-MM-DD_HHmmss` 目錄。

## 📅 日常使用完整流程

### 每日操作標準流程（SOP）

建議每天在同一時間執行（例如：台北時間早上 9 點）

```
[09:00] 前置檢查
   ↓
[09:05] 清空舊數據
   ↓
[09:07] Blave 數據抓取（10-15 分鐘）
   ↓
[09:22] 驗證數據完整性
   ↓
[09:25] 套用提示詞分析
   ↓
[09:35] 驗收結果與執行交易
```

---

### 步驟詳解

#### 步驟 1：前置檢查（3 分鐘）

確認以下項目：

| 檢查項 | 確認方法 |
|-------|---------|
| 在正確目錄 | `cd d:\自動化查詢幣種專案` |
| node_modules 存在 | `Test-Path node_modules` → True |
| .profile 存在 | `Test-Path .profile` → True |
| 無瀏覽器干擾 | 關閉所有 Edge/Chrome 視窗 |

#### 步驟 2：清空舊數據（2 分鐘）

**務必先做！** 避免舊數據影響判斷：

```powershell
npm run clear
```

確認看到「✅ 已清空 data/blave 目錄」

#### 步驟 3：Blave 數據抓取（10-15 分鐘）

執行主腳本：

```powershell
.\scripts\start_20260505_v1.ps1
```

等待完成，確認顯示「All tasks completed! (100%)」

#### 步驟 4：驗證數據完整性（3 分鐘）

檢查 10 項任務是否都完成：

```powershell
Get-ChildItem data\blave\ -Recurse -Filter task_result.json | ForEach-Object { Get-Content $_.FullName | ConvertFrom-Json | Select-Object task, ok }
```

應該看到所有 10 項任務的 `ok` 都是 `true`

#### 步驟 5：套用提示詞分析（10 分鐘）

選擇提示詞（推薦混合模式_穩健版），複製貼上到 AI 對話。

#### 步驟 6：驗收結果與執行（5 分鐘）

確認輸出包含：
- ✅ 立即執行表（最多 3 筆）
- ✅ 候選表
- ✅ 觀察清單
- ✅ 所有數據附來源與時間（UTC+8）
- ✅ 安全共振評分
- ✅ 明確的止盈止損點位

### 核心模組說明

#### 模組 1：capture_shared.mjs（共用函式庫）

這是所有抓取腳本的基礎，提供：

| 功能 | 說明 |
|-----|------|
| `formatTaipeiTimestamp()` | 產生 UTC+8 時間戳 |
| `launchContext()` | 啟動瀏覽器 persistent context |
| `checkBlaveLoginStatus()` | 檢查 Blave 登入狀態 |
| `createCaptureSession()` | 建立完整的抓取會話 |
| `saveFullPageScreenshot()` | 儲存完整頁面截圖 |
| `writeSummary()` | 寫入摘要檔案 |

**登入檢查機制：**
- 檢查是否有 profile/account 連結
- 檢查是否沒有登入按鈕
- 檢查是否有認證 cookie
- 若未登入，提示手動登入

## 🛠️ 故障排除與常見問題

### 問題 1：npm install 失敗

**症狀：**
```
npm ERR! code ENOENT
npm ERR! syscall spawn git
```

**解決方案：**
1. 確認網路連線正常
2. 若在公司網路，確認 proxy 設定
3. 試試看：
```powershell
npm install --registry=https://registry.npmmirror.com
```

---

### 問題 2：瀏覽器啟動失敗

**症狀：**
```
Target page, context or browser has been closed
```

**解決方案：**
1. 手動打開 Edge，造訪 https://blave.org/
2. 確認已登入
3. 關閉所有瀏覽器視窗
4. 重新執行腳本

---

### 問題 3：Blave 未登入偵測

**症狀：**
```
⚠️  偵測到未登入 Blave！
請在開啟的瀏覽器中手動登入 Blave
```

**解決方案：**
1. 在開啟的瀏覽器中登入 Blave
2. 登入成功後，回到 PowerShell 按 Enter
3. 若經常發生，確認 `.profile` 目錄權限正常

---

### 問題 4：數據抓取部分失敗

**症狀：** 某些任務的 `task_result.json` 顯示 `ok: false`

**解決方案：**
1. 單獨重新執行失敗的任務：
```powershell
npm run capture:blave:holder_concentration:overview
```
2. 檢查 Blave 頁面是否改版
3. 查看對應目錄下的 `page_full.png` 確認頁面狀態

---

### 問題 5：中文路徑錯誤

**症狀：** 路徑包含中文導致錯誤

**解決方案：**
1. 確認在根目錄執行：`d:\自動化查詢幣種專案`
2. 不要進入中文子目錄執行
3. 使用完整路徑執行腳本

---

### 問題 6：忘記所有命令

**解決方案：**
看這個文件！或者：
- 快速參考：`docs/06_快速參考小抄.md`
- 工作流程：`docs/05_完整工作流程指南.md`
- 執行步驟：`docs/00_執行步驟計劃書.md`

**建議：** 修改前先備份！

### 技巧 4：檢查數據新鮮度

確認數據是最新的：

```powershell
Get-ChildItem data\blave\ | Sort-Object LastWriteTime -Descending | Select-Object -First 5
```

檢查時間戳是否為今天。
