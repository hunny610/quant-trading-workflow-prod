# 交易提示字自動化系統 - 執行步驟計劃書

## 執行目標

使用 Blave 數據抓取 + 網路搜索的混合模式，產生可執行的交易策略表（UTC+8）

清除舊有的Blave 數據資料，才不影響後續的判斷

***

## 📋 整體流程總覽

```
階段 1: 使用者手動執行 Blave 數據抓取
    ↓
階段 2: 網路搜索補充數據 (5 個子任務，含強制保存)
    ↓
階段 3: 數據整合與分析 (3 個子任務)
    ↓
階段 4: 輸出最終結果
```

***

### 📝 1.4 幣種篩選機制說明

**幣種來源：**
系統會自動從以下 Blave 圖表中收集幣種清單：

1. **Blave 頂尖交易員-部位**：Top5 多頭、Top5 空頭

**提示詞篩選標準：**

- **非主流幣種**：可選取市值前 200
- **最終推薦數量**：3-5 隻主流幣種 + 2-3 隻非主流幣種（如果有好機會）

**你不需要手動指定幣種！** 系統會自動從 Blave 數據中收集候選清單，提示詞再進行綜合分析與篩選。

### 📝 1.5 Blave 登入驗證機制說明

#### 自動檢查流程（程式內建）
當執行Blave抓取腳本時，系統會自動：
1. 打開瀏覽器並跳轉到Blave頁面
2. 執行登入狀態檢查
3. 根據檢查結果自動處理

#### 自動檢查項目
- 是否有 profile/account 元素
- 是否沒有登入按鈕
- 是否有認證 cookie

#### 未登入時的處理流程
如果偵測到未登入：
1. ✅ **自動顯示提示**：
   ```
   ⚠️  偵測到未登入 Blave！
   請在開啟的瀏覽器中手動登入 Blave
   登入完成後，按 Enter 繼續...
   ```
2. ⏸️ **立即停止操作**，等待用戶
3. 👤 **用戶手動登入**：在打開的瀏覽器中登入Blave
4. ✅ **用戶確認**：登入完成後按Enter繼續
5. ▶️ **繼續執行**：程式繼續抓取數據

#### 已登入時的處理
如果已確認登入：
- 顯示：`✅ 已確認 Blave 登入狀態`
- 直接繼續執行抓取

#### 重要提醒
- **不需要預先手動登入**（雖然可以）
- 程式會自動打開瀏覽器讓您登入
- 登入完成後務必按Enter繼續
- 如果您想先預備好，可以先在一般瀏覽器登入Blave，再關閉所有瀏覽器視窗

## 階段 2：網路搜索補充數據（5 個子任務，含強制保存）

### ✅ 2.0 前置檢查（必須完成）

在執行本階段前，**必須先確認階段 1（Blave 數據抓取）已完成或跳過**：

| 檢查項                                    | 狀態 | 說明                                 |
| -------------------------------------- | -- | ---------------------------------- |
| \[ ] 確認 data/blave/ 目錄存在最新批次數據         | ⏳  | 查找最新 batch_YYYY-MM-DD_HHmmss/ 目錄（如有） |
| \[ ] 確認主要 Blave 圖表任務已完成（或跳過）         | ⏳  | 至少完成高優先級的 3 個任務，或標註 N/A |

**驗證命令**：

```powershell
# 先切換到正確目錄
cd quant-trading-workflow-test\20250506

# 檢查最新批次目錄
Get-ChildItem data\blave\ -ErrorAction SilentlyContinue | Sort-Object LastWriteTime -Descending | Select-Object -First 5

# 檢查所有 task_result.json（如有）
Get-ChildItem data\blave\ -Recurse -Filter task_result.json -ErrorAction SilentlyContinue | ForEach-Object { Get-Content $_.FullName | ConvertFrom-Json | Select-Object task, ok }
```

***

### 2.1 搜索最新價格

- BTC/ETH/SOL/XRP/BNB 最新價格
- 24h 漲跌、市值數據

### 2.2 搜索市場新聞

- 最新加密貨幣市場新聞
- 總體經濟事件（CPI、FOMC、非農就業等）
- ETF 流入數據

### 2.3 搜索關鍵事件（48h 內）

- 財報公佈
- 經濟數據發佈
- 重要新聞時間點

### 2.4 數據標註要求

- 每個數據必須附「來源名稱 + 資料時間（UTC+8）」
- 無時間戳 → 標註 N/A 並降低信心

### 2.5 保存搜索結果與分析報告（必須）

**⚠️ 重要：此步驟為強制執行，必須完成！**

將網路搜索結果和分析報告保存到 `data/search_out/` 目錄：

1. 使用 `data/search_out/範例_搜索結果保存格式_2026-05-05.md` 作為模板
2. 複製模板文件，重命名為您的標題（如：`市場分析報告_2026-05-05_125415.md`）
3. 將搜索結果和分析報告貼上並保存

**文件名格式（強制）**：`{標題}_{YYYY-MM-DD_HHmmss}.md`

**保存內容必須包含**：

- 網路搜索的價格數據（附來源與時間）
- 市場新聞摘要
- Blave 數據摘要（如有，無則標註 N/A）
- 完整的分析報告
- 交易策略表

### 3.2 選擇提示詞（請選擇一個）

1. **Blave 登入**：每次抓取前都會檢查，需要手動先在一般瀏覽器登入

3. **Blave 數據抓取由您手動執行**：請根據「階段 1」的指引，自己選擇並執行抓取腳本

7. **Blave 數據可選**：如果 Blave 抓取一直失敗，可以跳過，只用網路搜索模式

## 混合模式（Hybrid Mode）運作方式

1. **Blave 數據抓取由您手動執行**（🔴 最高優先級）
   - 您自己選擇並執行 Blave 數據抓取腳本
   - 每次抓取前自動清空 data/blave/ 目錄，確保數據最新
   - 如果批次抓取失敗，改用個別任務執行
   - 也可以選擇跳過 Blave 抓取，只用網路搜索
2. **網路搜索補充**（🟡 高優先級）
   - 用於獲取最新價格和新聞
   - 每個數據必須附「來源名稱 + 資料時間（UTC+8）」
   - Blave 數據不足時，網路搜索作為備援
   - 可以單獨使用網路搜索模式
